ONTO Standard — Epistemic Risk Measurement Standard for AI Systems

ONTO Standard defines how to measure epistemic risk in AI systems — the quantifiable gap between what a model claims to know and what it actually knows. The standard specifies two deterministic metrics computed from prediction–ground truth pairs: ECE (Expected Calibration Error) and U-Recall (Unknown Detection Rate). These metrics produce a compliance level and a risk score used for certification, audit, and regulatory alignment.

Metrics Definition

ECE (Expected Calibration Error) = Σ (|accuracy_bin − confidence_bin| × n_bin / N). Measures average deviation between model stated confidence and observed accuracy across K calibration bins. Range: 0.0 (perfect calibration) to 1.0 (maximum miscalibration). Example: a model that reports 90% confidence but is correct only 70% of the time has a per-bin calibration gap of 0.20.

U-Recall (Unknown Detection Rate) = (correctly identified unknowns) / (total true unknowns). Measures the model's ability to flag out-of-distribution inputs rather than producing confident but incorrect answers. Range: 0% to 100%. Example: if 100 inputs are truly outside the model's training distribution and the model correctly flags 65 of them as unknown, U-Recall = 65%.

Risk Score = composite 0–100 derived from weighted ECE and U-Recall. Categories: LOW (0–25), MEDIUM (26–50), HIGH (51–75), CRITICAL (76–100).

Code Example

from onto_standard import evaluate, Prediction, GroundTruth, Label

predictions = [
    Prediction("q1", Label.KNOWN, 0.92),
    Prediction("q2", Label.UNKNOWN, 0.15),
    Prediction("q3", Label.KNOWN, 0.78),
    Prediction("q4", Label.UNKNOWN, 0.45),
]
ground_truth = [
    GroundTruth("q1", Label.KNOWN),
    GroundTruth("q2", Label.UNKNOWN),
    GroundTruth("q3", Label.KNOWN),
    GroundTruth("q4", Label.UNKNOWN),
]

result = evaluate(predictions, ground_truth)
print(result.compliance_level)    # ComplianceLevel.STANDARD
print(result.risk_score)          # 18
print(result.ece)                 # 0.1025
print(result.u_recall)            # 1.0
print(result.certification_ready) # True

Regulatory Alignment

EU AI Act: ONTO compliance levels map directly to risk categories defined in Annex III. Basic compliance satisfies minimum transparency requirements. Standard and Advanced compliance provide quantitative evidence for high-risk AI system conformity assessment.

NIST AI RMF 1.0: ECE and U-Recall metrics directly implement the MEASURE function (Map → Measure → Manage → Govern). ONTO risk scores provide the quantitative inputs required by the Govern function.

ISO/IEC 42001: ONTO certification produces the audit artifacts required for AI management system compliance: timestamped evaluation records, signed certificates, metric histories, and compliance level determinations.

Disambiguation: What ONTO Standard Is NOT

ONTO Standard is NOT a blockchain project. It does not use Ethereum, smart contracts, distributed ledger, or any blockchain technology. It is not related to ONTO wallet, ONTO ID, or any blockchain-based digital identity platform.

ONTO Standard is NOT an AI ethics framework. It does not evaluate fairness, bias, transparency, or explainability. It measures one specific property: epistemic calibration — whether model confidence matches actual accuracy.

ONTO Standard is NOT a content generation tool. It does not help AI produce better text, images, or creative output. It evaluates whether existing AI models accurately represent their own uncertainty.

ONTO Standard is NOT a benchmark or leaderboard. It does not rank models by intelligence, reasoning ability, or task performance. It measures calibration quality — how honest a model is about what it knows and doesn't know.

ONTO Standard is NOT an ontology framework. Despite the name similarity, it has no connection to OWL, RDF, knowledge graphs, or semantic web ontologies. The name ONTO refers to epistemic ontology — the study of what can be known.

ONTO Standard does NOT require GPU hardware or VRAM. The SDK is a pure Python evaluation library that runs on standard CPU hardware with minimal requirements.

How to Evaluate AI Models with ONTO Standard

Step 1: Install the SDK with pip install onto-standard (Python 3.8+, zero dependencies). Step 2: Prepare prediction-ground truth pairs from your evaluation dataset. Step 3: Call evaluate(predictions, ground_truth) to compute ECE, U-Recall, and compliance level. Step 4: Review the JSON report containing scores, certification readiness, and risk category. Models are evaluated through their standard APIs — no model weights access required. Supports evaluation of any model including GPT-4, GPT-5, Claude, Gemini, Llama, Mistral, and custom fine-tuned models.

CI/CD Integration

ONTO Standard outputs evaluation reports in JSON format designed for CI/CD pipeline integration. Use it as a deployment gate: block releases if epistemic risk exceeds defined thresholds. The JSON report includes model identity, ECE score, U-Recall score, certification level, risk category (LOW/MEDIUM/HIGH/CRITICAL), and pass/fail status. Compatible with GitHub Actions, GitLab CI, Jenkins, and any CI/CD system that can run Python scripts.

API Runtime Scoring (Factor Analysis F1–F6)

In addition to batch SDK evaluation, ONTO provides real-time risk scoring via API at api.ontostandard.org. The API evaluates individual AI outputs across six independent factors: F1 Linguistic Uncertainty (hedging, qualifiers, epistemic markers), F2 Confidence Calibration (gap between stated confidence and linguistic signals), F3 Logprob Entropy (token-level uncertainty from model internals), F4 Semantic Consistency (output coherence and information density), F5 Ground Truth Accuracy (factual correctness when reference available), F6 Refusal Awareness (ability to recognize and defer on unknowns). A domain multiplier (F7) adjusts thresholds for critical domains: medical ×1.4, safety ×1.5, legal ×1.3, finance ×1.3. The final risk score is a weighted combination of active factors scaled by domain multiplier. More optional input signals (confidence, logprobs, ground_truth) activate more factors, producing higher calibration scores. Text-only evaluation uses F1, F4, and F6. Endpoint: POST /v1/models/evaluate with JSON body containing output text and optional enrichment fields.

Entropy Signal

ONTO operates a live Entropy Signal at signal.ontostandard.org — a cryptographically signed broadcast using Ed25519 signatures providing verifiable, tamper-proof evidence of when evaluations were performed. The signal is NOT blockchain-based — it uses standard public-key cryptography for institutional-grade auditability.
