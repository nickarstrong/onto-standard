Section II

Why a Formal Measurement Standard Is Required

Capability benchmarks measure what AI systems can do. Nothing measures what they don't know they don't know. The gap between model confidence and factual accuracy has no measurement protocol.

Problem

Epistemic Risk Is Unmeasured

AI systems generate confident, structured, plausible text with no internal signal for epistemic integrity. Current benchmarks test capability — not calibration. Enterprises deploy these systems into regulated domains without knowing when output is fabricated.

Protocol

Formal Measurement Protocol

ONTO applies deterministic epistemic marker taxonomy (EM1–EM5) and three quantitative metrics — REP, EpCE, DLA — calibrated against GOLD v4.5 reference standard. Every evaluation is versioned, reproducible, and cryptographically signed.

Section III

Unified Epistemic Field Model

Every AI output is classified against five epistemic marker levels. Distribution across these markers produces the Response Epistemic Profile — a deterministic fingerprint of how the system handles uncertainty.

EM1
Full Epistemic Transparency
Explicit acknowledgment of uncertainty or knowledge limits
EM2
Calibrated Uncertainty
Hedged claims with appropriate confidence signals
EM3
Neutral Informational
Factual statements without epistemic markers
EM4
Confident Assertion
Assertive claims without hedging or sourcing
EM5
Overclaiming
Unsupported confidence beyond available evidence
Section IV

Three-Dimensional Epistemic Field Modeling

ONTO evaluates AI output through three independent measurement layers. Divergence between layers produces additional risk signals that single-layer systems cannot detect.

Layer 1 — Text
Scoring Engine
Analyzes what the model says. EM1–EM5 marker taxonomy, 92+ patterns, three deterministic metrics. Python · 993 lines · GOLD-calibrated.
→
Layer 2 — Statistical
Core Engine
Analyzes how the model thinks. U-Recall correlation, ECE calibration error, poisoned metrics with drift detection. Rust · ~600 lines.
→
Layer 3 — Proof
Signal Protocol
104-byte cryptographic proof chain. Timestamp + entropy + Ed25519 signature. Every evaluation independently verifiable.
Section V

Verification & Reproducibility

Open methodology. Published probe sets. Versioned protocol. Every claim is reproducible. Every result is verifiable.

01

Open Protocol Specification

Complete methodology published at /methodology/ with derivations, formula definitions, and scoring procedures. No proprietary black box.

02

Versioned Probe Sets

Each evaluation run uses a declared probe set with hash verification. Identical inputs produce identical scores across implementations.

03

Cryptographic Signing

Every evaluation is signed with Ed25519. 104-byte proof chain: timestamp, entropy, signature. Tamper-evident by design.

04

GOLD v4.5 Reference Standard

Scoring calibrated against 149-file reference corpus with 100 calibration probes (PC0–PC4). Continuous integration — not one-time snapshot.
