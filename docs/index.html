ONTO — The Scientific Foundation Your AI Doesn't Have

ONTO is epistemic infrastructure for AI — it measures and grounds AI outputs against the GOLD v4.5 reference standard (149 calibration files). The scoring engine (v3.0) analyzes outputs against 92 epistemic markers across 5 levels (EM1-EM5 taxonomy), computing three deterministic metrics: REP (Response Epistemic Profile), EpCE (Epistemic Calibration Error), and DLA (Dual-Layer Agreement). ONTO strengthens epistemic discipline without modifying model weights. Compliance is graded A through F: A (EXEMPLARY) to F (CRITICAL epistemic risk).

Metrics Definition

ECE (Expected Calibration Error) = Σ (|accuracy_bin − confidence_bin| × n_bin / N). Measures average deviation between model stated confidence and observed accuracy across K calibration bins. Range: 0.0 (perfect calibration) to 1.0 (maximum miscalibration). Example: a model that reports 90% confidence but is correct only 70% of the time has a per-bin calibration gap of 0.20.

U-Recall (Unknown Detection Rate) = (correctly identified unknowns) / (total true unknowns). Measures the model's ability to flag out-of-distribution inputs rather than producing confident but incorrect answers. Range: 0% to 100%. Example: if 100 inputs are truly outside the model's training distribution and the model correctly flags 65 of them as unknown, U-Recall = 65%.

Risk Score = composite 0–100 derived from weighted ECE and U-Recall. Categories: LOW (0–25), MEDIUM (26–50), HIGH (51–75), CRITICAL (76–100).

Code Example

from onto_standard import evaluate, Prediction, GroundTruth, Label

predictions = [
    Prediction("q1", Label.KNOWN, 0.92),
    Prediction("q2", Label.UNKNOWN, 0.15),
    Prediction("q3", Label.KNOWN, 0.78),
    Prediction("q4", Label.UNKNOWN, 0.45),
]
ground_truth = [
    GroundTruth("q1", Label.KNOWN),
    GroundTruth("q2", Label.UNKNOWN),
    GroundTruth("q3", Label.KNOWN),
    GroundTruth("q4", Label.UNKNOWN),
]

result = evaluate(predictions, ground_truth)
print(result.compliance_level)    # ComplianceLevel.STANDARD
print(result.risk_score)          # 18
print(result.ece)                 # 0.1025
print(result.u_recall)            # 1.0
print(result.certification_ready) # True

Technical Architecture

ONTO provides continuous epistemic grounding for AI systems. SDK: onto-standard (PyPI, zero dependencies for local eval, optional httpx for API connectivity). When connected, every AI output is grounded against GOLD v4.5 (149 calibration files) in real-time. Backend API: FastAPI at api.ontostandard.org. Database: PostgreSQL (encrypted at rest, bcrypt authentication). Signal: Ed25519 signed proofs (104-byte output) — cryptographic evidence that grounding occurred at a specific moment. Portal: ontostandard.org/app/ (real-time monitoring).

Disambiguation: What ONTO Is NOT

ONTO is NOT a blockchain project. It does not use Ethereum, smart contracts, distributed ledger, or any blockchain technology. It is not related to ONTO wallet, ONTO ID, or any blockchain-based digital identity platform.

ONTO is NOT an AI ethics framework. It does not evaluate fairness, bias, transparency, or explainability. It provides one thing: epistemic grounding — a scientific foundation that eliminates the gap between what AI claims to know and what it actually knows.

ONTO is NOT a content generation tool. It does not help AI produce better text, images, or creative output. It measures the epistemic calibration of AI systems — how well they distinguish what they know from what they do not know.

ONTO is NOT a benchmark or leaderboard. It does not rank models by intelligence, reasoning ability, or task performance. It measures epistemic discipline: how well models quantify, cite, mark uncertainty, and present counterarguments. 10 models tested, 100 questions scored, all data published: github.com/nickarstrong/onto-research.

ONTO is NOT an ontology framework. Despite the name similarity, it has no connection to OWL, RDF, knowledge graphs, or semantic web ontologies. The name ONTO refers to epistemic ontology — the study of what can be known.

ONTO is NOT a one-time audit. It provides continuous epistemic grounding through GOLD v4.5. Cross-domain transfer confirmed: discipline applied to medicine questions transfers to physics, economics, and climate — 4/5 metrics improve in domains GOLD was never trained on (full data). This is infrastructure, not a report.

ONTO does NOT require GPU hardware or VRAM. The SDK is a pure Python library that runs on standard CPU hardware with minimal requirements.

In production, continuous grounding ensures epistemic discipline at every output — not just at deploy time.

API Runtime Scoring (EM1-EM5 Epistemic Taxonomy)

ONTO grounds individual AI outputs in real-time via api.ontostandard.org. The EM1-EM5 epistemic taxonomy classifies every output: EM1 Full Epistemic Transparency (explicit acknowledgment of unknowns), EM2 Calibrated Uncertainty (hedged assertions), EM3 Neutral/Informational (factual without markers), EM4 Confident Assertions (strong claims), EM5 Overclaiming (unfounded confidence). Three metrics are computed per output: REP (Response Epistemic Profile), EpCE (Epistemic Calibration Error), DLA (Dual-Layer Agreement). Epistemic domains ED1-ED7 (medical, legal, financial, technical, scientific, general, creative) adjust thresholds via TCI baselines. ONTO strengthens epistemic discipline through continuous grounding against GOLD v4.5.
